# oasis-website

Hugo source for the OASIS specification website (oasis-spec.dev).

## What this repository is

A Hugo site using the Hugo Book theme that renders the OASIS specification as a documentation website. The theme was chosen because it is a clean, single-CSS-file documentation theme that is straightforward for LLM-assisted maintenance — no complex JavaScript build pipeline, no framework abstractions.

## Relationship to oasis-spec

The specification content lives in a separate repository at github.com/jaimegago/oasis-spec. Content is pulled in at build time by a Go build script. This repository never modifies spec content — it only reads and renders it. The oasis-spec repository is the single source of truth for all specification text.

## Build script

The build script will live at `cmd/oasis-site-build/` (to be created in a later prompt). It is a Go program that fetches spec content from the oasis-spec repository and places it under `content/en/docs/` for Hugo to render.

## Known invariants

These rules must not be broken by any future session:

- **Single source of truth**: The oasis-spec repository (github.com/jaimegago/oasis-spec) is the sole source of specification content. Never hand-edit generated content under `content/en/docs/`.
- **No manual spec edits**: Content under `content/en/docs/` is generated by the build script. Manual changes will be overwritten.
- **No extra JavaScript**: The site must not include any JavaScript beyond what the Hugo Book theme ships with.
- **Hugo Modules for theme**: The theme is installed via Hugo Modules, not git submodules or vendored copies.
