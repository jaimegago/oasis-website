Contributions to OASIS happen through pull requests on the relevant repository. For changes to the specification itself — the core documents, profiles, and scenarios — submit pull requests to the [oasis-spec repository](https://github.com/jaimegago/oasis-spec). For changes to the website — styling, layout, landing page content, and tooling — submit pull requests to the [oasis-website repository](https://github.com/jaimegago/oasis-website).

Proposed changes to the specification should follow the RFC process documented on the [Governance](../governance/) page. Small fixes such as typo corrections and clarifications do not require an RFC and can be submitted directly as pull requests.

All contributors are expected to follow the [Code of Conduct](../code-of-conduct/). By participating in this project, you agree to abide by its terms.
